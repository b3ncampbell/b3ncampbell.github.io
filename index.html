Dropdown Menu

25th April 2017

Today I've added a dropdown menu to the mobile version of the site. It contains links to the other web pages I've created, and links that will jump you to each blog post. This makes navigation on a small screen much easier.

The menu is a styled div that is hidden by default, when the "Navigation" link is clicked the menu displays below the navbar. This is done with a JavaScript funtion that is called when the navigation link is clicked.

I've decided to hide the navigation link on the tablet/desktop versions of the site as they have all of the links in the right hand sidebar.

Centred Page

19th April 2017

This evening I've created a web page with a centred layout. The design is simple, but I wanted to make a responsive page with boxes and text that change their layout based on the size of the screen they are being displayed on.

I made the page responsive by using CSS and media queries, this way I could make each layout appear exactly how I wanted. For the next page I make I may use a flexbox layout to learn more about that, from what I know about it already I think it could be more efficient for creating a content-rich responsive website.

Tablet Layout

16th April 2017

Today I've added another media query to optimise the page layout when viewed on a tablet. Previously the site had the same layout for tablet and desktop viewing. Now when viewed on a tablet the site loses the side borders, with the main centre section taking up the full width of the screen. This makes the site much easier to read without any zooming in needed.

Now that I feel happy with the layout of the site on mobile, tablet, and desktop I want to start focusing more on learning JavaScript and jQuery. I will also start to make different web pages like I did with the image gallery page to get more practise creating page layouts.

Contact Form

3rd April 2017

Today I wanted to add a contact form to my site so that people can get in touch with me. After learning that this is mostly done using PHP (which GitHub Pages doesn't support), I started looking at other options.

First I looked at the forms that Google Docs can produce, but after testing them I felt that the design was too intrusive as they use a lot of styling that cannot be changed. After a bit of searching I came across Wufoo.

Wufoo are an online form building service which offer both free and paid solutions, with different pricing plans based on your needs. I have created a free account, which limits the number of forms I can create and how many entries they can accept per month. As there is little (no) traffic coming to my site right now that suits me just fine.

I may change the contact form to a custom made one once I start learning some PHP, but this Wufoo one will suffice for now.

I've also added a scroll to top arrow at the bottom of the page using jQuery. On the mobile site I've also included a scroll to bottom arrow on the navbar as there is currently no way of navigating to specific blog posts, making it easier to read the blog in chronological order.

Going Mobile

1st April 2017

After deploying my site on GitHub Pages and seeing how awful it was when being viewed on my phone I decided to focus on making it more mobile friendly. After all, I do most of my internet browsing on my phone now, and so do most of the people I know.

Whilst doing some research I found out that a lot of developers now create websites for mobile first, then use various styling techniques to make the site adapt to bigger displays - the opposite of what I had done.

I've spent the last day reworking my CSS file and making some layout changes to correct this. Now all of the default styling is for the mobile version of the site and I've used a media query to make it scale to a desktop version.

I am much happier with the layout of the site now, and by using the media query it makes it much easier to adjust the mobile site without affecting the desktop site and vice versa. For now I've only put in one media query, I may add a few more to better accommodate different display sizes.

Deployment!

Up until now this site only existed on my computer. Over the past couple of days I've been reading about the best way to get my work online, and have decided to use GitHub Pages as it is free, and I want to learn how to use Git also.

I loaded up the site on my phone to see how it looks as this is how a lot of people are viewing the web nowadays and... it's not great. Just a scaled down version of everything and the header has been moved out of line somehow!

Next up will be making the site responsive, and learning more about Git, as I've read that it's widely used and a valuable skill to have when working with other developers.
